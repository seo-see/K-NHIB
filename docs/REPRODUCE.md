# Reproduction Guide

Environment: Python 3.10, macOS/Linux. All commands run from the repository root.

## 1. Install

```bash
pip install -r requirements.txt
cp .env.example .env   # fill in ANTHROPIC_API_KEY, OPENAI_API_KEY, GEMINI_API_KEY
```

## 2. Generate model responses

```bash
python code/benchmark_pipeline.py
```
Writes raw per-(model, condition, case) results into `results/` (gitignored).

## 3. Aggregate & score

```bash
python code/aggregate_results.py
```
Produces a deduplicated master dataset plus summary metrics — accuracy, per-class
precision/recall/F1 (sklearn), and breakdowns by model/condition, cancer type, and class —
under `results/aggregated/`.

## Notes

- `results/` is not shipped in this repo; it is regenerated by the steps above.
- The `pdf` condition is the main analysis; `text_md` / `pdf_websearch` / `pdf_structure`
  are sensitivity analyses (see `prompts/README.md`).
- PDF parsing is isolated in `code/pdf_parser.py` (a pluggable `extract_text(pdf_path)`).
  It is used **only** by the text-input condition (`text_md`); the main `pdf` condition
  attaches the PDF natively and does not parse it. The study used **pdfplumber**; to
  experiment with a different parser, replace `extract_text` in that one file.
- This repo provides the **evaluation pipeline and scoring** so you can run any LLM on
  K-NHIB and obtain metrics. The manuscript-specific analysis and figure scripts
  (error taxonomy, sensitivity tests, bootstrap CIs, plots) are not included here.
